Initializing

git init or git clone <url>

Later add remote if “git init” was used

git remote add origin git@github.com:username/Hello-World.git
git branch --set-upstream master origin/master
git pull -u origin master

Merging

Keep all of a file (ours or theirs) during a conflict

Keep ours: git checkout --ours filename.ext

Keep theirs: git checkout --theirs filename.ext

then git add the files

Reset

unstage: remove files from index (tracking)

git reset HEAD

uncommit: go back before last commit, with files in uncommitted state

git reset --soft HEAD^

Delete branch

git branch -D bugfix or git push origin --delete <branchName>

Find what tag/release a commit is included with

git describe --contains <sha1>

Workflow

http://notes.envato.com/developers/rebasing-merge-commits-in-git/ http://nvie.com/posts/a-successful-git-branching-model/

Create feature branch

[master] git checkout -b feature

Pull changes from origin/master into master

[master] git fetch

[master] git rebase -p origin/master

Pull changes from master into feature

[feature] git rebase master

Merge feature branch into master

[master] git merge --no-ff feature

Delete feature branch when done

[master] git branch -d feature
